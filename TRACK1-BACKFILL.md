## 分轨原则（判定一个点该进哪轨）
- **机制 / 原理（怎么工作、在工艺里干嘛）→ 轨1**（到"理解流程"的深度）。
- **产业 / 价值（谁做、多大、壁垒、国产化、价值量）→ 轨2**；材料的**深度技术原理**作为"为什么难 = 壁垒"的铺垫，**跟轨2赛道页走**（和产业分析连贯成篇）。
- 判据：**"为看懂工艺" → 轨1；"为看懂生意 / 壁垒" → 轨2。**
- 只有当某点是"**理解制造流程的必经环节、轨1却缺**"时，才登记到本队列回补轨1。

## 协议（怎么用这个接口）
1. **投递**：轨2 遇到符合上面的轨1缺口 → 在下表加一行（一句话即可，标来源/哪页/补什么）→ 回到轨2，不跑题。
2. **回补**：集中处理时挑条目 → 在 `chip-explainer.html` 对应页补 → **真机渲染逐帧核对**（铁律）→ 发布 → 把状态改「🚀已发布」。
3. **状态**：⬜待补 · 🔧补中 · ✅已补未发 · 🚀已发布。
4. 回补**不一定要专门开轨1 session**——在轨2 session 里顺手补一项也行，改的是同一个 `chip-explainer.html`。
